CouchDB v9: Design Doc and Views

First Create the Database

Select the Create Database button to create a CouchDb database and insert one record. Select the Design button to create a Design Document. Design Documents are JSON files that contain CouchDb applications or the pieces out of which CouchDb applications can be constructed.

Now Read and Write to It

Choose Get Doc Names to get a list of the records that have been inserted. Put the name of one of the documents in the docName (the first) field, and press read.

Press the view button to see the output from the CouchDb view that we stored in the Design Document. In particular, this view shows the first and last name of one of the person records in our database

To create a new record, choose a new name for the document and put it in the Document Name field. Fill in your own data in the other fields, and press Insert to insert a new record.
